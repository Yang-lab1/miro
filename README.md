# Miro Prototype

Static frontend plus FastAPI backend for the Miro capstone concept.

## Local startup

The prototype should now be served over HTTP, not opened directly via `file://`.

1. Backend
   - `cd backend`
   - create a virtual environment
   - `pip install -e .[dev]`
   - copy `.env.example` to `.env`
   - `alembic upgrade head`
   - `uvicorn app.main:app --reload`
2. Frontend
   - from repo root, generate runtime config:
     - `npm run runtime:config`
   - serve the repo root with any static HTTP server, for example:
     - `python -m http.server 4173`
3. Open `http://127.0.0.1:4173`

For local HTTP smoke and proxy-backed QA, use `npm run smoke:http`.

## Deployment dry run

Use the standard post-deploy validation command:

- `npm run validate:online -- --frontend-url https://<frontend-domain> --backend-url https://<backend-domain> --expected-supabase-url https://<project-ref>.supabase.co`

## Prototype notes

- The prototype focuses on language and pragmatic coaching only.
- Pricing is now a dedicated top-level page, separate from Settings.
- Hardware Devices surfaces wearable pin status, version, transfer health, and sync logs as a support layer.
- Review Center merges simulation reports and device-synced meeting reviews.

## Folder guide

- Root: thin entry wrappers, package metadata, docs, and tests only
- `src/components`: reusable UI blocks such as the top nav and auth modal
- `src/data`: seed data and shared content dictionaries
- `src/pages`: route-level page templates
- `src/styles`: application styles
- `backend`: future backend implementation only
- `shared`: frontend/backend shared contracts, enums, and schemas
- `docs`: structured by `product`, `flows`, `api`, `architecture`, and `handoff`
- `scripts`: one-off inspection or QA scripts
- `artifacts/screenshots`: manual and Playwright screenshots
