# Docs Index

This folder is organized by document purpose so product, flow, API, and architecture work do not get mixed together.

## Structure

- `product`
  - product definition and scope
- `flows`
  - user flows, UI state maps, and cross-module behavior
- `api`
  - backend contracts, schema drafts, and integration-facing specs
- `architecture`
  - technical architecture, system boundaries, governance
- `handoff`
  - implementation status and project handoff notes
